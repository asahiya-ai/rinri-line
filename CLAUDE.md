# CLAUDE.md - rinri-line プロジェクト指示書

## プロジェクト概要
倫理法人会モーニングセミナーの文字起こしをClaude AIで要約し、LINEグループに毎週木曜9時に自動投稿するWebアプリ。
キャラクター「リンちゃん🐔」がにわとり口調で講話を紹介する。

## Claudeへの作業ルール

### コード修正時の自己チェックリスト
修正するたびに以下を必ず確認する：

- [ ] `systemPrompt` と `userMessage` の指示が矛盾していないか
- [ ] `charLimit` はハードコードせず変数を使っているか（`'・' + charLimit + '文字前後\n'`）
- [ ] GASのデフォルト値は `data.charLimit || 200`（150ではなく200）
- [ ] 要約プロンプトの変更時は `systemPrompt`・`userMessage` 両方を確認する
- [ ] Claude APIへの指示は「JSON形式のみで返す」旨を両方に含める
- [ ] 前処理フィルター（三宅耕平の発言除外）が全体を削りすぎていないか

### 変更後の再デプロイ忘れ防止
`gas/code.gs` を変更したら必ずユーザーに伝える：
「GASエディタに貼り付けて再デプロイしてください」

## 要約プロンプト（確定版・2026/05時点）

### systemPrompt（役割定義）
```
あなたはにわとり「リンちゃん」です。
倫理法人会モーニングセミナーの応援キャラクターとして、明るく親しみやすい口調で講話を紹介します。
借金・失敗・病気など具体的なエピソードは読み手が引き込まれるようにそのまま伝えます。
ただし自殺・死にたいなどの直接的な表現は「命の瀬戸際まで追い詰められた」などに言い換えます。
返答は必ずJSON形式のみで行い、説明文や前置きは一切不要です。
```

### userMessage の必須ルール
- 出力は本文部分だけ（冒頭あいさつ・締め文・【講話者さん】ヘッダーは含めない）
- 借金・失敗・病気・苦労などの具体的エピソードはそのまま書いてOK（読み手が引き込まれる）
- ただし「自殺」「死にたい」などの直接表現は「命の瀬戸際まで追い詰められた」等に置き換える
- 「この人の話、もっと聞きたい！」と思わせる引きのある書き方をする
- 語りかけ口調でテンポよく読めるように書く
- 文字数は `charLimit` 変数で指定（デフォルト200文字）
- 宣伝的な表現（「続きはセミナーで」など）は一切使わない
- 返答形式：`{"summaries": ["1人目の本文テキスト", "2人目の本文テキスト"]}`

## よくあるエラーと対処法

| エラー | 原因 | 対処 |
|--------|------|------|
| `要約の解析に失敗しました` | ClaudeがJSONでなく説明文を返した | `systemPrompt` と `userMessage` の矛盾を確認。両方に「JSON形式のみ」を明記する |
| 要約が空・短すぎる | 前処理フィルターが削りすぎている | `三宅耕平` の行除外ロジックを確認。次行まで飛ばしていないか確認 |
| `通信エラー (HTTP 4xx)` | GAS未デプロイ or URLが古い | GASエディタで「新しいバージョン」として再デプロイ |
| LINE送信されない | グループIDまたはトークン未設定 | スクリプトプロパティを確認。`testLineSend` でテスト実行 |
| 文字数がアプリ設定と連動しない | `charLimit` をハードコードしている | プロンプト内を `'・' + charLimit + '文字前後\n'` に修正 |

## LINE自動送信の仕組み

- 毎週木曜9時に `sendWeeklyLine` が自動実行（要 `setWeeklyTrigger` を一度手動実行）
- 自動送信対象：`LINE_SEND_VENUES = ['佐世保', '佐世保広報']`
- 佐世保広報は佐世保シートのデータを使い、佐世保広報グループIDに送信
- 送信順：テキスト → フライヤー画像
- 送信後に投稿済みフラグ（列G）が `TRUE` になる

## テスト方法

| 関数 | 用途 |
|------|------|
| `testSendLatest` | 佐世保の最新未投稿データをテストグループに送信（フラグ変更なし） |
| `testLineSend` | テキストのみのテスト送信（接続確認用） |
| `testSendKohoGroup` | 佐世保広報グループへのテスト送信 |
